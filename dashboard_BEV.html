Beverage Cost & Loss Driver Dashboard

ภาพรวมต้นทุนเครื่องดื่มต่อยอดขาย พร้อมไล่ดูว่าเครื่องดื่มตัวไหน — และ "หมวด" ไหน (เบียร์สด / สุรา / ไวน์ / น้ำอัดลม ฯลฯ) — คือสาเหตุที่ทำให้ของหายและ % ต้นทุนขยับขึ้น

กำลังโหลดข้อมูลล่าสุดจาก Google Sheet...

1 ภาพรวมรายเดือน — % ต้นทุนเครื่องดื่มเทียบยอดขาย

รายการ error (Lost ติดลบเกินครึ่งของ POS):

% ต้นทุนตามสต็อก vs ตามสูตร (POS)

%Variance เทียบยอดขายเครื่องดื่ม

2 สถานะการนับสต็อก — เดือนที่เลือก

เดือน:

รายการที่ไม่มีการจดสต็อก

ไม่มีข้อมูลทั้งต้นเดือน-1 และปลายเดือนเลยในเดือนนี้ — ตัวเลขของหายของรายการเหล่านี้ไม่น่าเชื่อถือ เพราะไม่ได้นับสต็อกจริง
เครื่องดื่ม หมวด

รายการที่สต็อกดี (ของหายไม่เกิน ±5 หน่วย)

มีข้อมูลต้นเดือน-1 และปลายเดือนครบ และของหายอยู่ในช่วง ±5 หน่วย — ถือว่านับสต็อกแม่นยำ ควบคุมได้ดี
เครื่องดื่ม หมวด ปริมาณหาย (Lost) มูลค่าหาย (บาท)

3 รายการเครื่องดื่มหายรายเดือน (Lost ตามสต็อก)

เดือน: หมวด:
ไม่มีการจดสต็อก = ไม่มีข้อมูลต้นเดือน-1 และปลายเดือนทั้งคู่ ดี = มีข้อมูลครบ และของหายไม่เกิน ±5 หน่วย
เครื่องดื่ม หมวด สถานะการจดสต็อก ราคา/หน่วย ปริมาณหาย (Lost) มูลค่าหาย (บาท) ต้นทุนตามสต็อก ต้นทุนตามสูตร (POS) %var (pp)

4 ไล่ดูสาเหตุ — เครื่องดื่มไหนทำให้ %ขยับ

เทียบช่วง: หมวด:
เครื่องดื่ม หมวด ราคา/หน่วย (ก่อน) ราคา/หน่วย (หลัง) Δ ราคา ปริมาณใช้ (ก่อน) ปริมาณใช้ (หลัง) Δ ต้นทุน (บาท) ผลจากราคา ผลจากปริมาณ สาเหตุหลัก

คำนวณจากข้อมูลระดับรายเครื่องดื่ม — มูลค่า = ปริมาณ × ราคา/หน่วยในรายงาน
%var (pp) = (ต้นทุนตามสต็อก − ต้นทุนตามสูตร) ÷ ยอดขายเครื่องดื่มรวมทั้งเดือน × 100 — รวมทุกเครื่องดื่มในเดือนนั้นจะเท่ากับ %Variance รวมของกราฟด้านบนเป๊ะ
หมวดเครื่องดื่มจัดอัตโนมัติจากชื่อรายการ (เบียร์สด/เบียร์ขวด-กระป๋อง/ไวน์/สุรา-สปิริต/โซจู/สาโทคราฟต์/น้ำอัดลม-น้ำดื่ม/อื่นๆ) — ถ้าจัดหมวดผิดสามารถแก้ไขฟังก์ชัน categorize_ ใน Apps Script ได้
